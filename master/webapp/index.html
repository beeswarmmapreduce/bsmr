Bee Swarm MapReduce

Welcome to Bee Swarm MapReduce demo. The server runs three different services — Master, Static, and PetriFS. Master is the key component that keeps of ongoing jobs. Static is an old fashioned web server that is used to serve the worker and console for demonstration purposes. PetriFS is a simple http-based file system that we use in the examples.

Example Use

  1. Install and run Google Chrome (known to work)
  2. Open the example application in one tab
  3. Go to the console in another tab and start a job
  4. Watch results appear in the data folder

You may wish to run a worker node separately to debug its behaviour.

In a real use scenario one should disable http access to the server entirely, and only allow WebSocket connections to the worker interface. Master also provides a console interface over the same WebSocket interface. It may be desirable to limit access to the console interface as well. Keep in mind that BSMR is a mere research prototype, and even if these measures are taken some loop holes will remain.
